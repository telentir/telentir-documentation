<div class="endpoint-grid">
  <section>

### Endpoint

| Method | Path |
| ------ | ---- |
| `PUT`  | `/api/objects/publish/{type}/{relatedId}` |

Associates an object with a publish slot so it can be surfaced in UI modules.

## Authorization

- Requires bearer token / API key with phone verification.

## Path parameters

| Param | Type | Description |
| ----- | ---- | ----------- |
| `type` | string | Logical collection name (e.g., `contacts`). |
| `relatedId` | string | Unique slug per published record. |

## Body

| Field | Type | Required | Description |
| ----- | ---- | -------- | ----------- |
| `objectId` | string | Yes | ID of the encrypted object to publish. |

## Response

HTTP `204 No Content`.

<details class="try-it">
  <summary>Try it</summary>

```bash
curl https://app.telentir.com/api/objects/publish/contacts/contact_123 \
  -X PUT \
  -H "Authorization: Bearer $TELENTIR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"objectId":"obj_123"}'
```
