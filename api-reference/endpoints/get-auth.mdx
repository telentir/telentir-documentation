### Endpoint

| Method | Path |
| ------ | ---- |
| `GET`  | `/api/auth` |

Use this endpoint to introspect the session tied to your cookie or bearer token. It mirrors `app/api/auth/route.ts`.

## Authorization

- Accepts Telentir workspace JWT or API key.
- Phone verification **not** required.

## Body

_No request body is accepted._

## Response

```jsonc
{
  "status": "ok",
  "auth": {
    "id": "user_123",
    "email": "ops@telentir.com",
    "firstName": "Ava",
    "lastName": "Nguyen",
    "phoneVerified": true,
    "twoFactorConfigured": true,
    "twoFactorPassed": true
  }
}
```

<details class="try-it">
  <summary>Try it</summary>

```bash
curl https://app.telentir.com/api/auth \
  -H "Authorization: Bearer $TELENTIR_TOKEN"
```
</details>

  </section>
  <aside>
    <h3>Examples</h3>
    <h4>cURL</h4>

```bash
curl https://app.telentir.com/api/auth \
  -H "Authorization: Bearer $TELENTIR_TOKEN"
```

    <h4>JavaScript (fetch)</h4>

```javascript
const res = await fetch('https://app.telentir.com/api/auth', {
  headers: { Authorization: `Bearer ${process.env.TELENTIR_TOKEN}` },
});
const data = await res.json();
```
