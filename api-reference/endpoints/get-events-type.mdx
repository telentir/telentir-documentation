### Endpoint

| Method | Path |
| ------ | ---- |
| `GET`  | `/api/events/{type}` |

Opens an SSE connection for key/object/publish events via `RedisEventStream`.

## Authorization

- Requires bearer token / API key with phone verification.

## Path parameters

| Param | Type | Description |
| ----- | ---- | ----------- |
| `type` | string | One of `key_created`, `key_deleted`, `object_created`, `object_updated`, `object_deleted`, `publish`, `unpublish`. |

## Query parameters

Any search params are forwarded as filter fields (e.g., `?objectId=obj_123`).

## Body

_Not applicable._

## Response

Content-Type `text/event-stream` with data payloads such as:

```text
event: message
data: {"id":"obj_123","user_id":"user_123","key_id":"key_123"}
```

<details class="try-it">
  <summary>Try it</summary>

```bash
curl https://app.telentir.com/api/events/object_created?objectId=obj_123 \
  -H "Authorization: Bearer $TELENTIR_TOKEN"
```
</details>
