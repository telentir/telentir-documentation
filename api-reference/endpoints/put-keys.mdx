### Endpoint

| Method | Path |
| ------ | ---- |
| `PUT`  | `/api/keys` |

Creates a new AES key for encrypting objects against a server.

## Authorization

- Requires bearer token / API key.
- Phone verification enforced via `objectAuth`.

## Body

| Field | Type | Required | Description |
| ----- | ---- | -------- | ----------- |
| `server` | string | Yes | ID of the server returned by `/api/servers`. |
| `key` | string | Conditional | Base64 key bytes when uploading an existing key. |
| `iv` | string | Conditional | Base64 IV; required when supplying `key`. |

## Response

- When Telentir generates the key, response is a string ID (`"key_123"`).
- When uploading, you receive the published key record:

```jsonc
{
  "id": "key_123",
  "server": "primary",
  "key": "a1b2c3...",
  "iv": "d4e5f6...",
  "deprecated": false
}
```

<details class="try-it">
  <summary>Try it</summary>

```bash
curl https://app.telentir.com/api/keys \
  -X PUT \
  -H "Authorization: Bearer $TELENTIR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"server":"primary"}'
```
